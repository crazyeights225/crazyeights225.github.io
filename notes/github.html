Github Notes.

Creating a new Repository

Instructions

Setup git:


//Install git:

~$ sudo apt-get install git


//Setup your username and email
~$ git config --global user.name "[YOUR USERNAME]"

~$ git config --global user.email "[YOUR EMAIL USED FOR THAT ACCOUNT]"

Create a new local repository

git init [A NEW REPOSITORY]

cd [A NEW REPOSITORY]

//Create a README
gedit README

git add README

//Commit the changes:

git commit -m "SOME MESSAGE"

On the github website create a new repository of the same name:

Commit the changes to the remote repository:

git remote add origin https://github.com/[YOUR USERNAME]/[YOUR REPOSITORY NAME].git

git push origin master
If you want to add more files:

//Add them to local directory commit changes, and them push to remote repository

git add HelloWorld.java

git commit -m "Added Hello World"

git push origin master

Updating a Repository

Instructions

//If you make changes to the remote repository without updating the local repository you
//must first fetch the changes:

cd [EXISTING REPOSITORY]
git fetch https://github.com/[YOUR USERNAME]/[EXISTING REPOSITORY]

//Or if the local repository does not exist:
git clone https://github.com/[YOUR USERNAME]/[EXISTING REPOSITORY]

Then to update an existing file:

git add .
git commit -m "Some Updates" 
git push origin master

To add new files:

git add [NEW FILENAME]
git commit -m "Added new file"
git push origin master
