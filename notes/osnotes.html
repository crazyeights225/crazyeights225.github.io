The Process Model

What is a process?
  • Entities running on top of the kernel
  • can be visualized as a tree hierachy
  • every process has a parent
Process vs System Calls
  • Running program on modern OS: processes
  • Request from a process to kernel: system call:
System Calls vs Function Calls
  • System Call: CPU must switch modes, jumps to a new address space
  • Function Call: CPU jumps to address of function (args, return address are pushed to stack)
What are the process states?
  • NEW - process being created
  • READY - process is waiting to be assigned to a processor
  • RUNNING - instructions are being executed
  • WAITING - waiting for event to occur (IO event, reception of signal)
  • TERMINATED - finished execution

What information is associated with each process?

The process id (PID) to identify process context, parent process ID, and process group ID.

Where is process information stored?

In /proc directory, there is one directory for each pid, and the file comm contains information about the process

What is the init process?
  • can build powerful facilities using process models
  • init is the ancestor of all processes
  • started as last step in kernel boot sequence
  • manages other processes in the operating system
  • pid of init is 1
Process Duplication

Cases for making a new process: running a new program
Duplicating an existing process - not the same as running a process a second time, variables and states are carried over, and the duplicate becomes the child and the original the parent.

Signals

What is a signal?

Something used for processes to communicate between each other. Signals cannot be used to count the number of occurences of an event. Signals indicate that at least one event of that type has occured.

What are signal handlers?

User defined functions to handle chosen functions

User defined signal

Signals from zombie children

  • When a child process terminates, the kernel also send a SIGCHILD signal to the parent
  • Child process calls back to kernel when it terminates, so kernel can inform the process' parent
  • Default behaviour is to ignore this signal
What is a Process Group?
  • a collection of one process or of related processes (they are usually related by one or more pipes)
  • at most one terminal that can be associated with a process group. There is a foreground process group, and a background process group (which is a process group that does not have access to a controlling process)
What is a Session?

A session is an environment for processes that is associated with one controlling terminal

Terminating and Reaping Processes

What does reap mean?

An act of parent process acknowledging that its child process has exited, thereby removing it from the parent table

What happens if a process terminates while it still has zombie children?

The init process reaps the zombies too. Then init is informed that is has zombie children. Init reaps the zombie processes

What happens if a parent process dies before its children?

The kernel makes init the parent of the orphaned children. When child process dies init recieves a SIGCHILD and reaps them.

What happens to a child process if no one waits for it?

When it dies it becomes a zombie (a process that has died but no one has waited for it)

What happens to a child process when a parent dies?

Another process becomes the parent.

Switching between runlevels

Each runlevel has a different (possibly overlapping) set of processes that are running. Also handles process termination scenarios

The C Process API

What are the Standard API calls to work with processes?
  • Kernel provides all these operations
  • fork() - creates a new process
  • exit() - terminates a running process
  • wait(), waitpid() - reaps terminated (zombie) child processes
  • execve() - loads and runs a program in a process
  • kill() - sends a signal to another process
Why call wait?
  • If no wait() the main proces terminates before the execve() command outputs
  • Adding a wait - the parent process waits for its child to terminate before it dies
wait vs waitpid
  • wait() - waits for child process to terminate
  • waitpid() - waits for a specific child to terminate, or all children in a process group
  • Both report an error if calling process has no children
Execve
  • Replaces the running process with something else
  • morphs into the command
  • does not return
Fork
  • the action to duplicate a process
  • based on return value of fork you can identify the parent process
  • Fork returns the PID of the child process to the parent
  • Fork returns 0 to the child

Disks and Diskblocks

What are disk sectors?

A sector is a physical spot on a formatted disk that holds information

What are disk blocks?

A block is a group of sectors that the OS can address. A block can have one sector or several sectors depending on the size of the disk. Each disk block has unique number. The file system always reserves a number of disk blocks (directory disk blocks) to store information about the files on the disk

What is the superblock?

A superblock is a record of the characteristics of filesystem, including size block size, empty and filled blocks, etx.

Mounted initially, the kernel reads information from it. Allows faster access to the file system, and for multiple file systems to be merged into a single tree

What is an inode?
  • Inodes stored in inode table (index number) containing information about a file.
  • A row corresponding to an index number contains: filetype, permissions, file size, location of file on disk, # of links, ...
What is lost+found?

lost+found is a directory used specifically be fsck, preallocates space for fsck to deposit files there. Files in that folder are files with an inode and no name, or an allocated inode that no file refers to. They are put in this foder to be deleted so they no longer take up space on the disk.

What does fsck do?

  • looks for damaged blocks (an allocated inode that no file refers to)
  • links the inode to lost+found directory using the inode number as filename
  • Cause could be unclean shutdown

Fragmentation and Defragmentation

What is fragmentation?

Defragmentation

Journalling Filesystems

What is a Journalling Filesystem?

A journalling filesystem is a filesystem that maintains a special file called the journal that is used to repair any inconsistencies that occurs as the result of inproper shutdown. They write metadata into a journal that is flushed to the HDD before each command returns. It guarentees the the structure of the filesystem is always consistent

3 Levels of Journalling (ext3)

  • Journal - writes data and metadata to the journal before committing it to the filesystem
  • Ordered - writes metadata data to the journal but commits data directly to the filesystem
  • Writeback - metadata is journalled, data is not, but metadata and data may be written in whatever order is best for performance
What is the reference count field in the inode table?

The reference count field in the inode table is the number of hardlinks presently pointing at a particular inode, when it is 0, it is an indication that the inode can be deleted, since nothing points to it anymore

What is an extent?

A range of contiguous physical blocks that can be reserved and addressed at once. Utilizing extents decreases the number of inodes required by a given file and significantly decreses fragmentation, and increases performance.

Linked List Allocation using Table (File Systems)

Take the pointer word from each disk block, and put it into a table in memory, chain terminated with special marker (FAT file allocation table)

What is a memory mapped file?

A segment of virtual memory that has been assigned a direct byte-for-byte correlation with some portion of a file or file like resource. Once present the correlation allows applications to treat the mapped portion as if it were primary memory

What is an access control list (ACL)?

refers to permissions attached to an output that specifies which users are granted access to that object and the operations that it is allowed to perform

What is bitmap?

One for data region, one for inode table, each bit is used to indicate whether the corresponding object/block is free (0) or in use (1)

Sequential vs Random Access (Files)

Sequential Access: system can only read files in order beginning to end
Random Access: bytes or records can be read in any order

Kernel Modules

What is a kernel module?
  • code that can be loaded and unloaded into the kernel on command
  • extends the functionality of the kernel w/o the need to reboot the system
  • Load a module: sudo insmod simple.ko
  • Unload a module: sudo rmmod simple.ko

How is a module unloaded?

The module data structure is marked as detected and it is unlinked from the list of kernel modules. Any other modules that are dependent on it have their reference list modified so that they have it as a dependent. All of the kernel memory that the module needed is deallocated.

Pages

What is a page?

A page, memory page, or virtual page is a fixed length contiguous block of virtual memory, described by a single entry in the page table. It is the smallest unit of data for memory management in a virtual memory OS

----> page (frame) unit of physical memory
----> page also a unit of logical memory. Size of page matches a page frame
----> Logical address is divided into a page selector and an offset into the page

Allocating Pages

page_alloc -> allocate n pages of virtual space, page table, mem_map data strucutre describes all physical pages, allocates a page in swap, multiple blocks use queue
2^(order) (1< contiguous pages are allocated, returns a pointer to first page

What is paging?

  • Used by virtual memory systems
  • Memory management scheme by which computer stores and retrieves data from secondary storage for use in main memory, retrieves data from secondary storage in same size blocks called pages

What is the page table?

Data structure used by virtual memory in an OS to store the mapping between virtual addresses and physical addresses. A page table entry is a frame number and an optional status.

Pages vs Sectors

A block is composed of one or more sectors but is no more than a page in size Assuming a 4K page size, a random read would require two 4K sector reads

Define: Page Fault

Exception raised by hardware when running program accesses memory page that is not currently mapped by the memory management unit (MMU) into virtual address of the process

How is Page Fault handled?

  • Os discovers a page fault has occured
  • finds out what virtual page is needed
  • Checks to see if address is valid
  • Performs disk transfer, loads frame, and update page table
  • faulting instruction backed up, faulting process is scheduled, OS returns to routine that is called
  • Routine rebounds registers, returns to user space to continue execution

How is the number of pages/page size defined?

1 << 12 = 4096 bits is one page, number of pages in program: 2^(order)(1 << order), contigous pages are allocated, returns address of first page

What is page replacement?

Algorithm: decides which memory pages to page out (swap out) or write to disk when a page of memory needs to be allocated

What is the swap?

portion of hard drive for virtual memory
-> a computer with virtual memory enabled will *swap* data to the HDD and back to memory as needed, thus, in effect, increasing the total system memory.

Does the kernel use virtual or physical addresses for its own data structures?

It would have to use both, since you cannot use virtual memory without knowing where it maps to.
-> There could be virtual addresses for the page table and virtual addresses using the page table.
-> kernel virtual memory is mappped into each process's virtual address space.

L2 CPU cache

second level, holds several megabytes of recently used memory words, access involves delay of 1 to 2 clock cycles

Concurrency

Define: mutual exclusion

The requirement that one process is in a criticial section that accesses shared resources, no other processes may be in a critical section that accesses any of those shared resources

What is a race condition?

Occurs when 2 or more threads can access shared data and they try to change it at the same time. Operations must be done in proper sequence to be done correctly.

Define Deadlock

When a set of processes are blocked because each process is holding a resource and waiting for another resource acquired by some process

Define spinlock

  • A lock that causes the thread trying to acquire it to simply wait in loop ("spin") while repeatedly checking if the lock is available
  • Thread remains active but is not performing useful task

3 Requirements to any solution to critical sections problem:

  • Mutual Exclusion: No 2 processes simultaneously in critical region
  • Progress: No process outside of a critical region may cause another process to block
  • Bounded: No process can wait forever to enter its critical region

Define: Atomic Operation

A function or action implemented as a sequence of one or more instructions that appears to be indivisble, that is no other process can see an intermediate state or interupt the operation. The sequence of instructions is guaranteed execute as a group, having no visible effect on system state. Atomically guarantees isolation from concurrent processes

4 Issues raised by concurrency for the OS

communication between process, sharing of and competing for resources, synchronization of the activities of multiple processes, Allocation of processor time to processes

Define Starvation:

Allocation of resources is unfair, something cannot proceed because it never gets the resources it require, even though resources may be available they are granted to something else

What is the basic requirement for the execution of concurrent processes?

The ability to enforce mutual exclusion, that is the ability to exclude other processes from a course of action while one process is granted that ability

Recovering from deadlock

  • Take resources if they preemptible
  • It can kill processes to free resources
  • Otherwise it can try to take snapshots of process states, and roll back on the state where the system is not deadlocked

How to exploit race condition?

There is a time between checking whether resource is available, and accessing the resource

Concurrency in kernel programming

Programs don't execute sequentially as user space applications do

Context Switching Steps:

  1. Saves enough registers on the stack to eventually return back to calling C function
  2. Stores the stack pointer in the current threads control block
  3. Load the stack pointer from the destination threads registers from its stack
  4. Restores the destination threads registers from its stack
  5. Returns to C in the destination thread
What are scheduling queues?

  • Processes in the Ready state are placed in the Ready Queue
  • Processes waiting for a device to become available are placed in device queues
  • A new process is initially in the ready queue, until it is selected for execution.
  • Once the process is assigned to the CPU: it could issue an IO request and be placed in the IO queue, could create a new subprocess unit for its termination, could be forcibly removed via interupt

  • segmentation gives user's view of process, which paging does not give. User's view is mapped into physical memory
  • Segment Table - maps 2d logical address into 1d physical address with base address, limit

What is segmentation fault?

Common condition causing program to crash, often associated with file named core. Segfaults are caused by a program trying to read or write to an illegal memory location

What are eBPF-bcc tools and how do they work?

  • allows users to exec semi-arbitrary code without having to load from a custom kernel module
  • kernel tracing tools - trace certain calls
  • they attach uprobes/kprobes to certain breakpoints, register callbacks which return data (the call return)
