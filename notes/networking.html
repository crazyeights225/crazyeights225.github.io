Networks: Types and Major Components

Definitions

Links

Link points of a network together. Ex. Wires, Fibre Optics, EM spectrum, etc

WAN

Wide Area Network, many owners, distributed control, approx. 10 km radius

MAN

Metropolitan Area Network, Large Area

LAN

Local Area Network, 1 owner 1 building, centralized control

Overhead

excess time, memory, bandwidth used by a network to perform a specific task

Circuit Switched

Traditional telephone model: a dedicated path through the network is established from source to destination. Resources are reserved for the duration of the session

Packet Switched

Postal Model: No reservation of resources. Packets advanced along a predetermined path, or created on-line, 1 link at a time. Packets stored temporarily at each node on the path.

Multiplexing

Sending multiple signals or streams over a communications link at the same time to form a complex signal. Recievers recover the seperate signals (demultiplexing)

What is the Cloud in the context of Networking

The magical cloud shape in the every network diagram that shows interconnected networks what is it?
A cloud is the unpredictable part of any network through which data passes between 2 end points. Clouds exists because any 2 points in a packet-switched network, the physical path on which a packet travels can vary from one packet to the next and in a curcuit network, the specific curcuit that is set up can vary from one connection to the other

What is a token ring?

a LAN technology, a communications protocol, uses a special 3-byte frame called a token that travels around a logical ring of workstations or servers. (in a ring or star topology)

Communication

Definitions

Hop-By-Hop (Point-to-Point)

Data and ACK/NAK transmitted to each hop on the route from source to destination. Messages are communicated across physical links electronically or optically. Only adjacent switches can communicate directly. Non-adjacent switches communicate cia intermediate switches.

End-to-End

Data transmitted to each hop on the route from source to destination, ACK/NAK only transmitted from destination to source.

Stop-and-Wait

SOURCE: transmits frame
DESTINATION: recieves frame, indicates willingness to accept another frame by sending back ACK to the frame just recieved.
SOURCE: must wait until it recieves ACK

Alternating Bit Protocol

Part of stop and wait. only works when channels are FIFO, appends a 1 bit sequence number to the message to stay in synch.
A -------> B m[0], 0
<------ ACK, 0

Problem with Stop and Wait

only frame at a time in frame, if propagation time is long relative to transmission time then the line will be idle most of the time.
For time t
Frame rate = 1/t then actual rate 1/t_frame+2*t_prop+t_ack

Sliding Window

Sliding window protocol is widely used, the sender continues the send action without recieving ACK from < w messages, where w is the window size, if no ACK recieved resends the w messages.

ENQ/ACK Process

  1. Sender send packet ENQ to determine if reciever is available.
  2. Reciever must answer either with ACK or NAK
    1. If sender recieves neither ACK or NAK it must assume ENQ was lost and sends replacement (make 3 attempts usually)
    2. If sender recieves a NAK to ENQ each time after 3 times it gives up and tries later
  3. After recieving an ACK sender sends data which are periodically acknowledged by the reciever
  4. Once all the data has been sent the sender sends EOT (end of transmission)

Errors

Error Types and Probabilities

Types of Errors:

Single Bit

Single bit changed, can effect larger blocks, more likely in parallel

Burst Errors

continous block of at least 2 error bits (starting with an error bit and ending with an error bit. Length of burst measured as the distance from the first corrupted bit to the last corrupted but

Error Recovery

Takes 2 forms: Detection and Correction. Detection is followed by retransmission. Correction most used at physical layer to produce bit pipe with low error rate

Probability and Errors

Number of errors in packet of size N, where p is the error rate: 

    	  \mbox{error rate} = \frac{\mbox{number of errors}}{\mbox{packet size}}
    	  
    	  \mbox{number of errors} = {p}\times{N}

Probability that there is exactly k errors, when error rate is p, and the packet size is N: 

    	  \Pr{(X=k)} = \left( \begin{array}{c} N \\ k \end{array} \right)\cdot{p^k}\cdot{(1-p)^{N-k}}

Probability that there is at least 1 error, when error rate is p, and the packet size is N

    	  \Pr{(X\geq{1})} = 1 - \Pr{(\mbox{no errors})}
      
    	  \Pr{(X\geq{1})} = 1 - (1 - p)^N

Expected Number of Errors

    	E(X)=	\sum {X\textsubscript{i}}Pr({X\textsubscript{i}})

Access Coordination

  1. A (B) listens to channel
  2. if channel is not busy
  3. then A (B) transmits packet
  4. if B (A) has not began transmission by time T_prop
  5. then A(B) is certain packet will reach B (A)
  6. else A (B) detects collision and retransmits

If user A is able to detect a collision it must occupy the channel for a time period of 2T_prop time units

How to measure time

Stations measure time T_A (T_B) from the beginning of thir own packet transmission to the time a collision occurs

Stations compare T_A and T_B with T_prop, A wins iff T_A < T_B, winning means loser remains quiet remains quiet until user completes transmission, after completing transmission winner remains quiet for 2T_prop time units to allow the user to capture channel

Efficiency

For each packet 2T_prop time is required to coordinate access
For bit rate R and packet length L then channel effeciency is: 

                  \frac{L}{L+2T\textsubscript{prop}R} = \frac{1}{1+\frac{2T\textsubscript{prop}R}{L}}
                
                  {T\textsubscript{prop}R} = \frac{{d}{v}}R
                    
                     \frac{T\textsubscript{prop}R}{L} = \frac{dR}{vL}

where d is distance, v is speed of medium, L is the packet length

n nodes contending for a channel. Node transmits during a contention with probability p, probability of success is equal to: 

                   \Pr{(Success)} = np\cdot{(1-p)^{n-1}}

Probability of success is maximized when p=1/n (take the derivative with respect to p

P_max is approx. e, and the average num of contentions until success is measured as the mean of a geometric distribution = e

Backoff Protocols

Probability distribution and a queue, there is a queue of stations (nodes) waiting to transmit, each station (node) keeps track of the number of attempts to transmit. Function p(x) defined in advance, known by all hosts, p(x) is the probability of transmission on the x-th attempt, Each station has var bck=0, each time it fails to transmit it increments bck

Binary Exponential Backoff

p(x) determine "aggressiveness", backlogged packet attempts with probability p(x) where x is the number of unsuccessful attempts. Exponential Backoff: p(x)=2^-x, Polynomial backoff: p(x)=1/(x+1)^k, where k is degree

CD

the hosts attempt detection of collisions, cuts down on unnecessary collisions

no voltage is detected, a transmission is initiated. During tranmission, station checks line for extremely high voltages that indicate collisions. If a collision is detected station wait a predetermined amount of time to clear (backoff) and sends the data again

Collision: Expected Backoff

Collision, each station waits 0 or 1 time slots and tries again -> If they pick the same random they collide again, waits 0, 1, 2, or 3 time slots-> Same random, waits 0, 1, 2, 3, 4, 5, 6, or 7 time slots -> ...

Expected Time

Contention period time t, this means they failed periods 0, 1, ... t-1, picks a random value 0..2^t-1, you must add all the waiting times for all the trials before t

                   \Pr{\mbox{(no collision)}} = \frac{2^t(2^t-1)}{(2^t)^2}=1-\frac{1}{2^t}

The waiting time for a success in t-th attempt will be: min{a_t, b_t}

Schemes: CSMA

1. 1-Persistent CSMA

Listens to channel to see uf anyone else is transmitting,if busy waits for idle, when idle transmits frame, if collision waits a random amount of time and starts over again.

1-Persistent means transmitter transmits with probability 1 whenever a busy line goes idle

2. Non-Persistent CSMA

Listens to channel, if idle transmits frame, if busy waits a random period of time and then repeats the algorithm

3. p-Persistent CSMA

Listens to channel, if busy wait until next slot and try again, if idle repeat until success or another node begins transmitting, transmits with pr = p, and defers with pr = 1-p, if another node begins transmitting node acts if there is a collision: waits random amount of time and starts again.

Ethernet CSMA-CD

CSMA improves over aloha, because it senses the carrier, however collisions involve entire packets, solution is to abort transmission when collision is detected
Stations senses channel, if idle: transmit, else: use CSMA strategy, if collision: transmit jamming signal and abort, schedule future transmission with backoff

Collision-Free Reservation Protocols: Basic Bit-Map

N stations labeled 0 to N-1, propagation delay is negligable.
each contention period has N slots, if station i has packet to send: it inserts a 1 bit in the i-th slot, no other station can transmit during that slot. After N slots, each station has complete knowledge of which stations wish to transmit, Stations begin transmitting in numerical order

Collision-Free Reservation Protocols: Binary Countdown

Assume all addresses are in binary, and of the same lenghth, Nodes write their bit starting highest to lowest to a register. They drop out as long as another node has written a 1 in current position, but continue otherwise

Splitting Algorithms

Split nodes into groups, each group assigned a slot, a collision can only occur within a group

Tree Splitting Algorithm

Nodes are leaves in a tree, first slot any node rooted at one may contend, if collsion in next slot only nodes in left subtree can contend, repeat

Triangulation

Process of determining location of a point by measuring angles to it from known points

The unknown distance x can be computed from:

                     d = \frac{x}{\tan\alpha} +  \frac{x}{\tan\beta} = x(\frac{1}{\tan\alpha} +  \frac{1}{\tan\beta})

Therefore:

                     x = \frac {d}{(\frac{1}{\tan\alpha} +  \frac{1}{\tan\beta})}

Another way to measure distance

Two sensors x distance apart: A sends a signal to B in medium1, B responds to A in different medium2, Both A, B measure the roundtrip time T

Let v1, v2 be the propagation speeds in media medium1, medium2 respectively, let T1 be time A to B, T2 be the distance B to A, they both measure RTT = T1 + T2, and since v = d/t, (T1 and T2 are unknown) then:

                      v\textsubscript{1}T\textsubscript{1}=v\textsubscript{2}T\textsubscript{2}

Use substitution to get:

                      T\textsubscript{1}=\frac{v\textsubscript{2}T}{v\textsubscript{1}+v\textsubscript{2}}
                
                      T\textsubscript{2}=\frac{v\textsubscript{1}T}{v\textsubscript{1}+v\textsubscript{2}}

Distance Vector

Based on Bellman-Ford Algorithm

Initializes distance to the source to 0 and all other nodes to infinity.
For all edges, if the distance to the destination can be shortened by taking the edge, the distance is updated to the lower value.
At the ith iteration when edges are scanned, the algorithm finds all shortest paths of at most length i (in at most edges)
Uses the idea of shortest path routing, handles topology changes. Forms routing table as an array of tuples (destination, nexthop, distance)
To send a packet to a given destination, it is forwarded to the process in the corresponding nexthop field in the tuple
In a graph with n nodes, the distance vector D for each node i contains n elements D[i, 0] through D[i, n-1], where D[i, j] defines the distance of node i from node j
Initially:
D[i,j] = { 0 if i=j, 1 if j is a neighbor of i, infinity otherwise
Each node j periodically broadcasts its distance vector to its immediate neighbors
Every neighbor i of j, after recieving the broadcasts from its neighbors, updates its distance vector as follows:

                     D[i, k]=min\textsubscript{j}\{w[i, j]+D[j, k]\}, \forall{k\ne{i}}

When a node j or link crashes, some neighbot k of it detects the failure, and sets the corresponding distance D[j, k] to infinity
When a new node joins, or existing node is repaired the neighbor detecting it sets the corresponding distance to 1.
Following this, the distance vectors are corrected, and the routing table is recomputed

Distance Vector Routing Algorithm

  1. Nodes initialize the routing tables
  2. Every node sends its vector to all its directly connected neighbors
  3. Every node after recieving this information updates its distance vector

Nodes maintain their information locally. No node has all the information
Convergence: the process of getting consistent routing information.

Routing algorithm in rounds, in each round, each node: transmits its vector to all its neighbors, recieves vectors from all its neighbors, updates its vector on the basis of what it recieves

Calculating Maps and Shortest Paths

First give a map of the network to each node by: 1) Determining the neighbors of each node, 2) Distributing the information for the map, 3)Creating the map
Second: each node independently runs an algorithm over the map to determine the shortest from themselves to every other node in the network
Nodes maintain 2 data structures: tree containing list of completed nodes, and list of candidates, Algorithm starts with both structures empty and adds the first one to the node itself.

Repeat:

Adds to candidate list of all nodes which are connected to the node just added to the tree (except those already there)
Of the nodes in the candidate list, moves to the tree (attaching it to the closest neighbor node already there) the one which is the closest to any of the nodes already in the tree
Repeat as long as there are any nodes left in the candidate list
This procedure ends with the tree containing all nodes in the network, with the node on which the algorithm is running as the root of the tree. The shortest path from that node to any other node is indicated by the list of the nodes one traverses to get from the root of the tree, to the desired node in the tree

Link State Protocol

  1. Each router is responsible for the information on its neighbor

  3. LSP is transmitted to all other routers
  4. Each router (now knows a complete map of the network) and computes routes to each destination

Spanning BFS Tree (Breadth First Search Tree)

BFS Algorithm

Breadth First = Level Order

Breadth First Search Tree: A BFS tree T of a graph G is a spanning tree of G such that for every node of G, the tree path is a minimum-hop path to the root

  1. mark root r
  2. mark all neighbor vertices that are one hop away from r
  3. mark vertices that are one hop away from these neighbors which are one hops away from these neighbors, which are 2 hops away from r
  4. Repeat

BFS Distance Computation

Root (source) has distance d(s)=0, the other nodes have distance d(i)=infinity
At the kth step (starting at k=0), all nodes i at distance d(i)=k are examined, and any neighbors j with d(j)=infinity have have their distance set to k+1.
The process halts when step k finds no such neighbors, d(j) is then the length of the shortest path from s to j, or d(j)=infinity if there is no path. It is only suited for unweighted graphs it ignores edge weights.

Route Calculation in LSP: Dijkstra's Algorithm

Pick fixed node r to start at, M is set of nodes incorporated so far by algorithm. Add route cost node to M, Update min cost routes to r. Repeat

Algorithm computes weights of paths, not paths themselves

Bridge has ports corresponding to each of the LANS it is connected to.

A spanning tree of a network is a subnetwork that is a tree (ie contains no cycles), and includes all of the nodes of the network
If the edges of the network are weighted a minimum weight spanning tree is one with minimum sum of the edge weights.

Prim's Algorithm

P is a set of nodes in tree, and Di is min weight edge from node i to node to a node in P
Initially P={1} and Di=w(i,1) if (i, 1) exists, infinity otherwise:
1. Find i not in P such that Di is minimum
2. P = P or {i}
3. For j not in P, Dj = min{Dj, w(j,i)}
4. Go back to 1

Kruskal's Algorithm

1. Sort the edges of G in increasing order (by weight)
2. Consider edges in order and add to tree if the result does not form a cycle

Exponential RTO Backoff

When a TCP sender timesout on a segment it must transmit. Since timeout is probably due to network congestion it is best to reuse the RTO value. If no segments are retransmitted Jacobsons algorithm works well. When segment timeout and need to retransmitted the RTT values in Jacobsons algorithm are discarded

A new suggestion is: Do not use the measured RTT for a retransmitted segment to update SRTT and SDEV, calculate the backoff RTO when a retransmission occurs, use the backoff RTO value for succeding segments until an ACK arrives for a segment that has not been retransmitted, when such an acknowledgement is recieved Jacobsons algorithm is used to compute future RTO values.

Complicating Factors

TCP connections are multiplexed so that each connection is allocated only a fraction of available capacity. TCP connections will involve hops across multiple networks. The propagation delay resulting is the sum of the immediate propagation delays. Bottlenecks occurs because data rate bigger than available data rate in any one of the hops. Throughput is reduced by lost or corrupted segments.

Future Flow Rate

Depends on bottlenecks. Causes need for retransmission timer management. Done with 3 algorithms.

RTO (Retransmission TimeOut) Backoff

When TCP sender timesout on a segment it must retransmit. Since timeout is probably due to conjestion it is best to reuse RTO value. This may cause problems when many TCP connections are active. The follwing strategy is used by TCP source:
1. Increase RTO value each time same segment is retransmitted, (this is backoff process).
2. TCP source will have to wait longer time for a second retransmission.
3. Update according to: RTO <- qRTO (usually q=2)

Window Management

Size of TCP window can have critical effect on avoiding conjestion in TCP. Four approaches: Slow Start, Dynamic Window Sizing on Congestion, Fast Retransmit, Fast Recovery

Slow Start

Gradual expansion of window size, constrained by awind={credit, cwind}, where awind is allowed segment in the window, cwind is congestion window used during startup, credit is amount of unused credit granted in most recent ACK.
Steps: 1. For new connection: cwind=1, 2. Each time an ACK is recieved cwind increases by 1 for each incoming packet.
Its growth is exponential, bc next time cwind ACKs arrive the new value will be 2cwind.

Basic Rule

Every positive ACK, cwind is increased by 1/cwind, and every time lost packet detected cwind is halved.

Dynamic Window Sizing on Conjestion

More conservative growth rate than slow start, applies when timeout occurs.
set slow start threshold: setfresh= cwind/2, setcwind=1 and perform slow start process until cwind=setfresh, here cwind is increased by 1 for every ACK recieved. For cwind less than equal to setfresh increase cwind by one for each round trip time.

Fast Retransmit

retransmission timer RTO is set, usually somewhat greater than estimated RTT (SRTT) because RTO calculated on the basis of a prediction of next RTT estimated from past values. If delays in source or dest fluctuate then the estimated RTT may be smaller than actual. The dest may not ack each segment individ but cummulatively.
As consequence when segment lost TCP may be slow to retransmit.
Fast Retransmit Rules: if TCP entity recieves a segment out of order it must immediately issue an ACK for the last inorder segment that was recieved. TCP continues to repeat ACK with each incoming segment until missing one arrives to fill hole in its buffer. When hole is full TCP sends cummulative ACK.

Fast Recovery

When TCP entity retransmits a segment it assumes segment was lost even though it has not yet timed out on this segment. Assume ACKs eventually arrive indicating the ack of lost segment fast recovery technique is used: retransmit lost segment, cut cwind in half, proceed with linear increase of cwind.
